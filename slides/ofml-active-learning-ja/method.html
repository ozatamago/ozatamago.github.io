Object-Flow Machine Learning

プロトコル構造を保存した代理モデリング

問題設定では、真の実行過程をプロトコル依存の未知写像 \(\{F_{j,p}^\dagger\}\) で、OFML 仮説クラス（共有モジュール \(g_{\theta_j}\)・合成予測器 \(\Eval(p;\theta;x)\)・パラメータ不確実性 \(\Pi_n^\theta\) と終端関数への pushforward \(\Pi_n^{G}\)）を §3.3 で定義した。本節はその合成アーキテクチャ・学習・閉ループ選択の具体を箱で示す（形式的定義は §3.3、一致性は 理論）。

プロトコル構造付き代理モデル

合成実行規則（アーキテクチャ）

問題設定 §3.3 の共有モジュール \(g_{\theta_j}:\Zcal_j\to\Scal_j\) と観測器読み出しヘッド \(h_q\)（\(\theta\in\Theta\) は操作モジュールと観測器ヘッドの全学習成分を束ねる）を、真の実行と同じ DAG の位相順序で合成する：

\[ z_v^{\theta}(x) = \iota_{p,v}\!\left((s_u^{\theta}(x))_{u\in\Pa_p(v)},\, x\right),\qquad s_v^{\theta}(x) = g_{\tau_p(v)}\!\left(z_v^{\theta}(x)\right). \]

観測予測は観測器ノードの読み出しをまとめた \(\Eval(p;\theta;x)=\left(h_q(s_{v(q)}^{\theta}(x))\right)_{q\in\Obs(p)}\in \Ycal_p^{\mathrm{obs}}\)。真の実行 \(F_{\tau_p(v),p}^\dagger\) を学習可能成分 \(g_{\tau_p(v)}\) に置き換えた形が OFML の代理モデルであり、§3.3 の仮説クラスの実行規則を具体化したものである。主設定の終端読み出し \(h_{q^{\mathrm{term}}}\) は端末スカラー測定、1-C の中間観測器ヘッドは学習可能である。

OFML の中心：操作型ごとのパラメータ共有 ＝ 転移の機構

同じモジュール型 \(M_j\) が異なるプロトコル・異なる位置・異なる出現回数で現れても、同一の予測成分 \(g_{\theta_j}\) を用いる。このパラメータ共有こそがプロトコル間知識転移の唯一の機構である。プロトコル全体を単一のカテゴリ特徴や平坦な特徴ベクトルとして扱うブラックボックス代理モデルでは、この操作単位の共有構造は明示されない。

↑ 例：M にホバーすると、4 本すべてのプロトコルの M が同一成分として強調される。source で M を学べば、未知プロトコルの M にそのまま再利用できる。

観測データからの学習

終端教師のみ（中間トレースは使わない）

OFML は操作ごとの局所入出力 trace や中間潜在状態の教師信号を仮定しない。訓練時に直接与えられるのは測定器ノード出力をまとめた観測値だけであり、すべての予測成分 \(\{g_{\theta_j}\}\) はプロトコル全体を通した観測予測誤差から同時に学習される。

経験リスク最小化

データ集合 \(\Dcal_n=\Dcal_{\mathrm{src}}\cup\Dcal_{\mathrm{tar},n}\) に対して

\[ \mathcal L_n(\theta) = \sum_{i} \ell_{p_i}\!\left(y_i,\ \Eval(p_i;\theta;x_i)\right) + \lambda\,\Omega(\theta), \qquad \widehat\theta_n\in\operatorname*{arg\,min}_{\theta\in\Theta}\mathcal L_n(\theta). \]

\(\ell_{p_i}\) は観測空間 \(\Ycal_{p_i}^{\mathrm{obs}}\) 上の損失（MSE・負の対数尤度など）、\(\Omega\) は正則化。すべての \(g_{\theta_j}\) はプロトコル全体を通した終端観測誤差から同時に学習される（操作単位の局所教師は用いない）。

識別可能性は自明でない

この目的は操作ごとの局所写像を直接教師あり学習するものではない。観測出力だけから各モジュール成分をどこまで識別できるかは自明でなく、理論で局所入力被覆・目標関連識別可能性として扱う。

不確実性付き予測と閉ループ選択

パラメータ不確実性 \(\Pi_n^\theta\)（§3.3）の実装

§3.3 のパラメータ不確実性 \(\Pi_n^\theta\) を、主実装では \(M=5\) の深層アンサンブル（seed 違い）で近似する（MC-dropout / HMC は代替の \(\Pi_n^\theta\)、点推定のみなら \(\Pi_n^\theta=\delta_{\widehat\theta_n}\)）。

予測分布＝終端関数への pushforward

候補 \((p,x)\) の予測分布は §3.3 の pushforward \(\Pi_n^{G}=(T_p)_{\#}\Pi_n^\theta\)、すなわち任意の可測 \(B\subseteq\Ycal_p^{\mathrm{obs}}\) について \(\Pi_n^\theta(\{\theta:\Eval(p;\theta;x)\in B\})\)。予測平均・共分散を

\[ \mu_{n,p}(x)=\E_{\theta\sim\Pi_n^\theta}\!\left[\Eval(p;\theta;x)\right],\qquad \Sigma_{n,p}(x)=\Var_{\theta\sim\Pi_n^\theta}\!\left(\Eval(p;\theta;x)\right). \]
獲得関数と閉ループ選択

予測分布 \(\Pi_n^{G}\) から獲得関数 \(\alpha_n(a)=\alpha(a;\Pi_n^{G},\Pcal_\star)\)（MaxVar／IVR／SUR）を計算し、次実験を \(a_{n+1}\in\operatorname*{arg\,max}_{a\in\Acal_n}\alpha_n(a)\) で選ぶ（§3.5 の行動 \(a=(p,x,q)\)・単位コスト）。観測 \(y_{n+1}=\Eval^\dagger(p_{n+1};x_{n+1})+\xi_{n+1}\) を得てデータを \(\Dcal_{\mathrm{tar},n+1}\) に追加し、逐次再学習する。

実験での具体化

本実験では語彙は \(\{A,B,C,D,M\}\)＋埋め込み \(\text{embed}\)＋端末観測器 \(O\)。直列プロトコルなので実行は s = embed(x); for op in steps[:-1]: s ← g_op(s); y = obs(s) に帰着する（非残差 \(s_v = g_{\theta_{\tau_p(v)}}(z_v)\)）。主実装の不確実性は \(M=5\) の深層アンサンブル、MC-dropout / HMC は代替の \(\Pi_n^\theta\)。比較手法とアーキテクチャは Experiments の設定を参照。

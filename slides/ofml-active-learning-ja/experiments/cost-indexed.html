中心結果 · 順方向転移＋ターゲット適応

図 2 — コスト指標での順方向転移とターゲット適応

本節の中心結果。単一のコスト指標曲線が実験全体の物語を統合する：横軸は target ラベルの累積取得コスト \(b\)、縦軸はマクロ NRMSE（各 target の自スケールで正規化した誤差）。\(b=0\) が順方向転移（主張 1）、\(b>0\) が能動的ターゲット適応（主張 2）、そして曲線が張り付く水準が記述子エイリアシング床（terminal-only／順序盲の限界、主張 1）である。3 つの代理モデルを同一の均衡 Sobol スケジュールで比較し、構造だけを切り分ける。

実験目的（主張の要約）

同一の固定スケジュール下で、コスト指標マクロ NRMSE 曲線 \(\eps_m(b)\) が (i) \(b=0\) の順方向転移、(ii) \(b>0\) のターゲット適応、(iii) 予算到達時 \(b=B\) の固定予算誤差を、記述子エイリアシング床を基準として同時に示すことを確認する。合成的な OFML-Ens は転移も適応も最良で、順序を無視する count 記述子 Shared-Ens はエイリアシング床に張り付く（＝平均予測）。さらに OFML-Ens ではモデル駆動獲得（MaxVar / IVR）がランダム取得に勝つ。

実験設定

合成 GP 合成場ベンチマーク gpcomp_factory.build_bench(depth, bench_seed)。2 つの深さ：depth-3（操作 \(\{A,B\}\)）と depth-5（\(\{A,B,C,M\}\)）。各深さの target は操作数が同じで順序だけ異なる2 本にして順序盲エイリアシングを露出させる。3 つの代理モデルはすべて種のみ 5 メンバの深層アンサンブル（bootstrap なし）で容量一致（hidden_dim=32）：Independent-Ens（target ごとに独立学習、コスト 0 は source 平均へフォールバック）、Shared-Ens（操作 count 記述子ベース、順序盲）、OFML-Ens（合成ネット、latent_dim=10）。各コスト \(b=0..B\) でゼロから完全再学習。構造スケジュール（均衡 Sobol、両 target 同一 evidence）で図 2a、OFML-Ens のみ random / maxvar / ivr で図 2b。予算 \(B=200\)、12 bench seed（200–211）を主統計単位に。観測ノイズ \(\sigma=0.02\)。

※ 前版からの主要な是正：(1) NRMSE 化（各 target の自スケールで正規化。生 RMSE は廃止）。(2) 参照集合リーク修正 — IVR/MaxVar の参照プールを test/取得プールと互いに素な Sobol ストリーム（scramble A/B/C）から構成（以前は test の部分標本 = 透過的リーク）。(3) seed 交絡の解消 — 1 本の固定関数（SEED=200）の 12 反復ではなく、bench_seed で真のベンチマーク自体を 12 回引き直す（task/pool/noise/model/acq ストリームを分離し、モデル間比較を task で対応化）。

詳細な設定（seed 分類・手続き・QC・計算）
seed 分類. bench_seed（200–211 の 12 本）が主統計単位で、真のベンチマークモジュール自体を引き直す（＝12 個の関数ドロー。1 関数の 12 反復ではない）。派生ストリームを分離：task=bench_seed、pool=bench_seed×7+1、noise=bench_seed×7+2、model=bench_seed、acq=bench_seed×7+3（モデル間比較を task で対応化＝ペア化）。Sobol は sob=bench_seed×100003＋固定オフセット（test +11 / pool +101 / ref +201 / 構造スケジュール +301）。

手続き. 各累積コスト \(c=0..B\) で train＋取得済みの和集合に対しゼロから完全再学習（warm-start なし）。構造スケジュール＝2 target を交互に取る固定の均衡 Sobol interleave で、全モデルが同一の source 行・同一の target evidence を見る（＝純粋な構造比較）。獲得（OFML-Ens のみ）：maxvar＝標本分散最大、ivr＝rank-1 Cohn ALC スコア \(\sum_R \mathrm{cov}_{rc}^2/(\mathrm{var}_c+\sigma^2)\)（\(\sigma^2=0.02^2\)）。選択点は最近傍で取得プールから除去。

統計. 図 2a の帯は ±1 SE（std/√n）、図 2b は seed 対応パーセンタイルブートストラップ（n_boot=10⁴、95%、win_frac 併記）。

QC / リーク対策. test / pool / ref / スケジュールは互いに素な Sobol ストリーム、test はノイズ無（train-on-test なし）、完全再学習でウォームスタート汚染なし。ベンチマーク工場は bench_seed=200 で既存ベンチと byte 単位一致を assert。144 record すべて status=ok（失敗 0）。

計算. 合計 CPU 時間 ≈199 h（ProcessPool 24 worker で実時間 ≈8.3 h）。OFML-Ens の 201 コスト完全再学習が支配的。

考察 — 主張をどう支えるか

1 枚の曲線が 2 主張を束ねる。\(b=0\)（順方向転移）：合成的な OFML-Ens が最良で、順序盲の Shared-Ens は最初からエイリアシング床＝主張 1。床（張り付き）：count 記述子は同数・異順序 target を原理的に区別できず、terminal-only の識別限界を示す＝主張 1。\(b>0\)（適応）＋獲得利得：target 終端を直接能動観測すると誤差が床を大きく下回り、情報獲得がランダムに勝つ＝主張 2。エイリアシング床という共通の基準線を導入したことで「どのモデルがいつ床を割るか」を一目で読める。

順方向転移の詳細（線形 s0 パリティ・順序・count）は 図 1a／図 4／図 5、識別可能性の直接監査は Φ/Ψ 監査、terminal-only の失敗集合は 表 1（C1–C6）を参照。

※ 限界：図 2b の獲得利得は OFML-Ens のみ（Independent/Shared-Ens は設計上 acquisition 比較なし）。Independent-Ens のコスト 0 は定数（source 大域平均）で学習転移ではない。図 2a の帯は ±1 SE でブートストラップ CI ではない。エイリアシング床・sup 誤差は 48×48 dense グリッド近似、マクロ NRMSE は 256 test 点上の評価。

理論との対応

コスト指標曲線は主定理の構造をそのまま可視化する。\(b=0\) の順方向転移は識別可能性（rank 条件 \(\ker\Phi\subseteq\ker\Psi\)、命題 4）に依存。エイリアシング床は転移診断の残差（仮定 8：C1–C5）に対応。\(b>0\) で床を割る収束は、全台 \(\lambda\) 上の雑音付き \(\IVR/\SUR\) による潜在関数分散の全域消失（定理 2）と、pushforward で特化した target active correction（定理 3）・rescue（命題 5）。図 2b の獲得利得は、獲得関数のランキングではなくボトルネック（被覆／容量）の診断として読む。

2
転移が失敗しても、構造の優位で能動学習が誤差を早く下げる

b=0 で識別が破れても、target 終端を能動観測すれば誤差は床を割り、合成的 OFML は非構造ベースラインより速く適応する。

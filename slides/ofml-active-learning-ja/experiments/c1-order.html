主張 1 · 合成は転移を助ける（order 識別）

図 4 — order 識別による順方向転移（非可換性）

主張 1 の中核。非可換な操作列（order）を source から held-out 順序へ順方向転移できるかを、held-out 順序でのマクロ NRMSEで測る。線形合成 OFML-linear は各操作の行列を source から同定し、行列積の順序として held-out 順序を厳密に再構成する（マクロ NRMSE ≈0.017）。訓練順序を陽に見た合成ネット OFML-Ens は訓練参照では小さいが held-out 順序で崩壊し、操作 count しか見ない Shared-Ens は同 count・異順序を区別できずどの順序でも失敗する。これは主定理の rank 条件 \(\ker\Phi\subseteq\ker\Psi\)（命題 4(i)）そのものの検証である。

実験目的（主張の要約）

非可換な操作合成において「source が順序方向を張れば（識別可能なら）held-out 順序へ順方向転移できる」という主張 1 を、held-out 順序でのマクロ NRMSEで直接検証する。線形合成 OFML-linear は各操作の \(2\times2\) 行列を source から同定し、held-out 順序を行列積の順序として厳密に再構成する（全 4 順序で NRMSE ≤0.026、マクロ 0.017）。訓練順序を陽に見た合成ネット OFML-Ens は訓練参照 A B C M O では小さい（0.076）が held-out 順序で崩壊（マクロ 1.256）、操作 count しか見ない順序盲の Shared-Ens は同 count・異順序を区別できずどの順序でも失敗する（マクロ 2.169）。「合成そのもの」ではなく「識別可能性を伴う合成」が転移の鍵であることを切り分ける。

実験設定

合成アフィン線形族（非可換操作 \(\{A,B,C\}\) ＋終端 \(M\)・線形読み出し \(O\)）。source は順序を張る 12 本：単操作 3 本（A M O, B M O, C M O）、両順序の 2 操作ペア 6 本（A B M O/B A M O, A C M O/C A M O, B C M O/C B M O）、非 held-out の全順列 3 本（A B C M O, B C A M O, C A B M O）。これらは各操作行列と主要な非可換方向を励起し、真の \(\theta\) で \(\ker\Phi\subseteq\ker\Psi\) を満たす。target は訓練参照 1 本 A B C M O とheld-out 3 本 A C B M O・B A C M O・C B A M O（source に一度も現れない残り 3 順列。count は訓練参照と同一で順序だけが違う）。各プロトコルは per_proto=40 点で学習し、n_test=120 点（ノイズ無・真値）で評価。3 モデルはすべて同一の source/target evidence を見る（純粋な構造比較）。12 bench seed を主統計単位に、指標は全てマクロ NRMSE。観測ノイズ \(\sigma=0.02\)。

※ held-out 3 順序は訓練参照とcount が完全に同一で順序だけが違う（順序盲エイリアシングを露出させる設計）。OFML-linear は決定論的な線形同定＋合成なので seed 変動は数値・観測ノイズ程度、OFML-Ens／Shared-Ens は種のみ 5 メンバの深層アンサンブル（容量一致 hidden_dim=32）。生 RMSE は使わず、各 target の自スケールで正規化した NRMSE に統一。

指標はすべてマクロ NRMSE。各 target のプールドスケール \(s_\star\) で正規化し、target 集合上で平均する：\[ \eps_\pi=\sqrt{\mathrm{mean}_x\,(\mu_\pi-f_\pi)^2\,/\,s_\star^2},\qquad \NRMSE_{\mathrm{macro}}=\frac{1}{|\Pcal_{\mathrm{tgt}}|}\sum_{\pi\in\Pcal_{\mathrm{tgt}}}\eps_\pi,\qquad s_\star=\sqrt{\tfrac{1}{|\Pcal_{\mathrm{tgt}}|}\textstyle\sum_\pi \mathrm{mean}_x\big((f_\pi-\bar f)^2\big)}. \] ここで \(\mu_\pi\) は順序 \(\pi\) の予測、\(f_\pi\) は真値、\(\bar f\) は全 target 平均。\(\eps_\pi\) が各順序の NRMSE（図・表の各セル）、その target 平均がマクロ NRMSE。\(s_\star\) は共通 48×48 dense グリッド上で計算する。

詳細な設定（seed 分類・手続き・QC）
seed 分類. bench_seed（12 本）が主統計単位で、真のアフィン線形族（操作行列群＋読み出し）自体を引き直す。派生ストリームを分離：task=bench_seed、pool=bench_seed×7+1、noise=bench_seed×7+2、model=bench_seed（モデル間比較を task で対応化＝ペア化）。Sobol は sob=bench_seed×100003＋固定オフセット（test +11 / grid +301）。OFML-linear は閉形式同定のため seed 依存は観測ノイズ経由のみ。

手続き. 全モデルが同一の 12 本 source 行と同一の target evidence（訓練参照 A B C M O の 40 点）を見る。OFML-linear は source から各操作行列を最小二乗同定し、target 順序を行列積として合成して評価（held-out 順序はゼロショット：target evidence 不要）。OFML-Ens／Shared-Ens は source＋訓練参照で学習し、held-out 順序を予測。

QC / リーク対策. test はノイズ無（train-on-test なし）、held-out 3 順序は source・訓練参照のいずれにも現れない（順序リークなし）。count は 4 順序で同一に固定し、順序以外の交絡（操作集合・深さ）を排除。全 record status=ok。

読み方. held-out 順序での NRMSE 差だけが「順序（非可換性）を識別できたか」を測る。count 同一・順序のみ相違という設計により、Shared-Ens の失敗は原理的下限（順序盲の床）を与える。
実験結果

OFML-linear が全 4 順序で最良（各順序の最小）。 held-out 3 順序でも NRMSE ≤0.026 に留まり、訓練で一度も見ていない順序を行列積として再構成する＝順序（非可換性）を識別している。OFML-Ens は訓練参照 A B C M O では 0.076 と小さいが、held-out 順序で 0.635–2.691 に崩壊（マクロ 1.256）＝順序の外挿ができない。Shared-Ens は count が同一の 4 順序を区別できず、訓練参照でも 1.417 と高くマクロ 2.169（順序盲の床）。

考察 — 主張をどう支えるか

held-out 順序での成否が主張 1 の核を画定する。OFML-linear は source が順序方向を張るとき（\(\ker\Phi\subseteq\ker\Psi\)）非可換合成をゼロショットで転移する＝主張 1。OFML-Ens の崩壊は、同じ合成構造でも識別可能性を保証しない当てはめは未観測順序へ外挿しないことを示す（合成は転移の十分条件ではなく、識別可能性が鍵）。Shared-Ens は count 記述子が原理的に順序を消すため、同 count・異順序を区別できない＝terminal-only／順序盲の識別限界＝主張 1。count は同一・順序のみ相違という設計により、この差は「順序（非可換性）を識別できたか」だけを純粋に測る。

識別性の直接監査（真の \(\theta\) の \(\Phi/\Psi\) 階数）は Φ/Ψ 監査、線形 s0 パリティは 図 1a、count エイリアシングは 図 5、コスト指標での位置づけは 図 2を参照。

※ 限界：OFML-linear の near-zero 誤差は線形同定が厳密に効く合成族での結果で、残差は数値・観測ノイズ \(\sigma=0.02\) 由来。held-out 順序は count 一致・順序のみ相違に限定（順序以外の交絡は排除）。OFML-Ens の崩壊値は seed 分散が大きく（特に A C B M O）、点推定は 12 seed 平均。図の縦軸は 0–3 にクリップ（OFML-linear の棒は視認のため最小高 3px で描画、実値はラベル・表を正）。

理論との対応

本実験は主定理の rank 条件をそのまま検証する。source が順序方向を張り \(\ker\Phi\subseteq\ker\Psi\) が成立するとき、線形合成は held-out 順序へ厳密に転移する（命題 4(i) の rank 条件そのもの；同値に \(\mathrm{row}(\Psi)\subseteq\mathrm{row}(\Phi)\)、条件数 \(K=\sigma_{\max}(\Psi)/\sigma_{\min}^{+}(\Phi)\) が誤差増幅を支配）。OFML-Ens の held-out 崩壊は、識別可能性を課さない合成当てはめが命題 4 の保証域外にあることの実証である。仮に識別が破れる順序（source が張らない順列）でも、target 終端を直接能動観測すれば終端応答一致性を回復できる（定理 3・命題 5）。

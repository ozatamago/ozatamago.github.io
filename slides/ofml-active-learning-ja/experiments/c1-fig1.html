主張 1 · 合成は転移を助ける

図 3a — 線形族 s0 パリティ：合成的転移の存在証明

構造化された terminal-only 転移の存在証明。source プロトコルで学んだ操作知識を、同じ操作を別の順序で合成しただけの未知 target（s0）へゼロショットで再結合できるかを問う。合成ネット OFML-linear は s0 をマクロ NRMSE 0.042（自スケール正規化）で解き、潜在再結合が平均として近完全であることを示す。一方、順序を見ない count 記述子 Shared-Ens は同数・異順序の 2 target を区別できず、両者の平均を予測してエイリアシング床（NRMSE ≈1.3）に張り付く。ただし OFML-linear の 3 リスタート・アンサンブルは同一解へ収束し予測分散がほぼ消える（2σ 被覆 0%）＝過信であり、これは正直に明記する。

実験目的（主張の要約）

プロトコルをカテゴリや操作数（count）記述子ではなく共有操作モジュールの合成として扱えば、source で学習した操作知識を未知の target へ再結合できる — これが主張 1「合成は転移を助ける」の存在証明である。s0 は識別残差 \(r^{\mathrm{id}}=0\)（\(\ker\Phi\subseteq\ker\Psi\)、Φ/Ψ 監査参照）を満たす識別可能な再結合なので、理論上は転移が成功するはずである。本図はその予言が経験的に成立すること（合成ネットが近完全、順序盲ベースラインは床）を、マクロ NRMSE・2σ 被覆・予測 std の 3 指標で確認する。

実験設定

各手法を共通 source 9 本（端末 \(y\) のみ観測）で学習し、s0 の target 2 本をゼロショットで予測する。s0 の 2 target は操作の多重集合が同一で順序だけが逆（A B C M O と C B A M O）で、順序盲エイリアシングを露出させる。source は各 30 点（per_source=30、学習ノイズ \(\sigma=0.02\)、計 270 点）、test は各 target 120 点（n_test=120、ノイズ無＝真値）。統計単位は 12 seed。グリッド（図 3a）の列は 8 手法だが、本ページの表は代表 3 モデル — 容量一致の合成 OFML-linear、順序盲 count の Shared-Ens、合成アンサンブル OFML-Ens — に絞って構造だけを切り分ける。指標はすべてマクロ NRMSE（各 target の自スケールで正規化。生 RMSE は用いない）。

※ 正直な注記：OFML-linear の不確実性は 3 リスタート・アンサンブルの分散で与えるが、s0 は識別可能で最小化子が実質一意のため 3 メンバがほぼ同一解に収束し、予測 std ≈0 になる。結果として 2σ 被覆が 0%（＝誤差はほぼ 0 なのに区間が縮退して真値を「外す」）という過信が生じる。平均予測は近完全でも較正は破綻している点を強調する。

指標はすべてマクロ NRMSE。target \(p\) の自スケール \(s_p=\sqrt{\operatorname{mean}_x(f_p-\bar f_p)^2}\) で正規化し、s0 の \(P=2\) target 上でマクロ平均する。較正指標として 2σ 被覆と平均予測 std を併記する。\[ \NRMSE=\sqrt{\frac1P\sum_{p}\frac{\operatorname{mean}_x(\mu_p-f_p)^2}{s_p^2}},\quad \mathrm{cov}_{2\sigma}=\operatorname{mean}_{p,x}\mathbf{1}\!\left[\,|\mu-f|\le 2\hat\sigma(x)\,\right],\quad \bar\sigma=\operatorname{mean}_{p,x}\hat\sigma(x). \] \(\mu\) は予測平均、\(f\) は真値、\(\hat\sigma\) はアンサンブル予測 std。\(\mathrm{cov}_{2\sigma}\) の名目水準は約 95%（近似正規）で、これを大きく下回れば過信、大きく上回れば保守を意味する。

実験結果

s0 パリティ — 合成が近完全に再結合する。 容量一致 OFML-linear はマクロ NRMSE 0.042 ± 0.005 で s0 をほぼ完全に解く（対角線に張り付く）。合成アンサンブル OFML-Ens は 1.158 ± 0.043、順序盲 count の Shared-Ens は 1.300 ± 0.048 でエイリアシング床に張り付く（異順序 2 target を同一予測＝両者の平均）。OFML-linear の圧倒的優位は、s0 が識別可能（\(r^{\mathrm{id}}=0\)）で、真の線形操作族に容量一致した合成ネットのみが潜在再結合を実現できることを示す。

考察 — 主張をどう支えるか

線形族で OFML-linear が s0 をマクロ NRMSE 0.042 で解くことは、操作順序・合成が本質的で、かつ構造が識別可能なレジームでは合成が非合成ベースラインを大きく凌駕するという存在証明であり、主張 1 を直接支える。s0 が識別可能（\(r^{\mathrm{id}}=0\)、\(\ker\Phi\subseteq\ker\Psi\)）であることが転移成功の前提であり、その階数関係を真の \(\theta\) で直接測る監査が Φ/Ψ 識別可能性監査である。対照的に、順序盲 count の Shared-Ens は同数・異順序 target を原理的に区別できずエイリアシング床に張り付く＝terminal-only 記述子の限界（主張 1）。

本図は過大主張を避け、正直な限界も併記する：OFML-linear の 2σ 被覆 0% は識別可能レジームでアンサンブル分散を不確実性に用いることの落とし穴（過信）で、能動観測やベイズ較正なしには不確実性が信頼できない。合成の鋭い分離を較正込みで示す設定は 図 3b・図 3c、コスト指標での順方向転移＋適応の統合は 図 2を参照。honest failures（未知操作 s1、三体 s7 など）は terminal-only の識別限界として 主張 1で扱う。

理論との対応

s0 パリティの成功は主定理の識別可能性条件に対応する。合成が転移する存在証明は、線形 rank 条件 \(r^{\mathrm{id}}=0\iff\ker\Phi\subseteq\ker\Psi\)（命題 4）に依拠し、s0 は \(r^{\mathrm{id}}=0\) で識別可能なので転移が成功する（数値的な直接監査は Φ/Ψ 監査）。識別可能なとき知識転移が経験的に意味を持つことは 仮定 8（転移診断 C1–C5）に対応する。識別が破れる target では、終端を直接能動観測すれば終端応答一致性を回復できる（定理 3・命題 5）。

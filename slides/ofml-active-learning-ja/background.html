Background — Introduction

なぜプロトコル構造を保存した代理モデルが要るのか

論文の Introduction を、動機 → 既存手法とのギャップ → 現実世界との観測構造のギャップ → 提案（OFML）という流れで、図とインタラクションを交えて追う。本文は Introduction を原文のまま用いている。

① 動機

閉ループ自律実験では、プロトコル自体が探索の途中で変わる

動機（Introduction 第1段落）

近年、AIとロボットの統合により、仮説生成、実験実行、観測、モデル更新を一つのサイクルとして回す閉ループ自律実験が現実的になりつつある[SDL][mobile][GPT5-lab]。生命科学や実験化学の研究現場では、研究者は固定された実験空間を一度だけ探索するのではなく、観測結果に応じて試薬や測定条件、さらには操作順序など、実験プロトコルそのものを変更しながら探索を進める[bofn]。したがって、閉ループ自律実験における代理モデルには、単一プロトコル内での予測精度だけでなく、異なるプロトコル間で知識を再利用し、少数の実験から次の実験を選ぶデータ効率が求められる。

この2つの要求（プロトコル間の知識再利用・データ効率）が、本研究の代理モデル設計を貫く軸になる。

② 既存手法とのギャップ

完全な機構シミュレータと単一ブラックボックスの「間」に余地がある

既存手法とのギャップ（Introduction 第2段落）

既存のアプローチは、この要求に対してそれぞれ異なる限界を持つ。完全な物理・化学シミュレータは、プロトコル内部の機構を明示的に扱えるが、未知パラメータ、未観測中間状態、実験系ごとの境界条件を含むウェット式実験では構築が容易ではない[E-CELL][whole-cell]。一方、ガウス過程やニューラルネットワークに基づくブラックボックス代理モデルは、少数データでの最適化に使いやすいが、プロトコルを識別子や平坦な特徴量として扱うと、操作順序、操作共有、測定器の違いといった構造を失う[reaction-BO]。また、大規模言語モデルは仮説生成や文献知識の利用には有用であるものの[Coscientist][ChemCrow]、それ単体では校正された事後不確実性や、プロトコル構造に沿った獲得関数を自然に与えるものではない。このため、完全な機構モデルと単一ブラックボックス代理モデルの間に、プロトコル構造を保存した代理モデリングの余地が残されている。

③ 現実世界との観測構造のギャップ

代理モデルが受け取るのは「終端出力」だけ — どの操作の誤差か分からない

下のトグルで「中間観測」を切り替えると、終端観測のみの制約が生む誤差帰属のあいまいさと、各操作の直後に観測器モジュール O_A・O_M（並列・非破壊の側枝）を置いたときの解消を体感できる（これが後の主張 1（1-C 中間観測）の核心）。

プロトコル実行：x → A → M → O → y（潜在 s は d 次元のまま本流を流れる）
終端観測のみ：A・M 直後の状態は記録されず、終端 y の誤差をどの操作に帰属させるかが不定。研究者は実際には途中経過を暗黙に把握しているが、それはデータとして残らず、代理モデルには渡らない。
観測構造のギャップ（Introduction 第3段落）

この間隙は、実験科学に特有の観測構造によってさらに明瞭になる。人間中心の実験では、実験者はプロトコルに明記されない手技、目視確認、環境調整、経験的判断を通じて、操作の途中経過を暗黙に管理している[tacit]。しかし、閉ループ自律実験において代理モデルが受け取るデータは、多くの場合、測定器を通じた終端出力に限られる。中間状態が直接観測されない場合、各操作に対応する局所モデルを単独で教師あり学習することはできず、終端誤差をどの操作に帰属させるかも自明ではない[DeepGP]。ロボット化によって実験操作の反復性が高まるほど、この終端観測制約は、操作単位の知識転移を支配する代理モデル設計上の中心問題として前景化する。

④ 提案：Object-Flow Machine Learning

プロトコルを「実験対象・操作列・測定器」の構造化入力として扱う

提案（Introduction 第4段落）

本研究では、この問題に対して Object-Flow Machine Learning（OFML）を提案する。OFML は、実験プロトコルを単一の識別子や平坦な特徴ベクトルとして扱うのではなく、実験対象、操作列、測定器からなる構造化された入力として扱う。具体的には、同じ種類の実験操作が異なるプロトコルに現れる場合に、その操作に対応する予測写像のパラメータを共有し、プロトコル全体の予測を終端観測に対して学習する。この設計により、OFML は、完全な物理・化学シミュレータほど詳細な機構知識を要求せず、単一のブラックボックス代理モデルよりもプロトコル内部の操作構造を保存する、構造化代理モデリングを実現する。

全体像（source で学習 → 未知プロトコルへ再結合 → target で能動補正）は Overview のアニメーション図を参照。

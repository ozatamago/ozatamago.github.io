Problem setting

終端観測付きプロトコル構造化代理モデリング問題

OFML が解く対象を定義する。実験プロトコルの構造、真の潜在過程、測定器を通じた観測、および source から target への転移目的を、要点ごとに箱で示す（背景の地の文は補足のみ）。学習される代理モデルは Object-Flow Machine Learning で導入する。

プロトコル集合と操作語彙

操作語彙とプロトコル集合

共通する操作語彙を \(\Mcal=\{M_1,\ldots,M_m\}\) とする。各 \(M_j\) は分注・混合・加熱・培養・反応・読み出し準備など、再利用される操作型を表す。プロトコル集合を \(\Pcal\)、観測済みの source を \(\Pcal_{\mathrm{src}}\subseteq\Pcal\)、転移先の target を \(\Pcal_{\mathrm{tar}}\subseteq\Pcal\) とする。

型付き有向非巡回グラフ（DAG）と入力構成写像

各プロトコル \(p\in\Pcal\) は型付き DAG \(p=(V_p,E_p,\tau_p,\iota_p)\)。\(V_p\) はノード集合、\(E_p\) は依存関係、\(\tau_p:V_p\to\{1,\ldots,m\}\) は操作型を割り当てる。ノード \(v\) の親集合を \(\Pa_p(v)\)、外部設計変数を \(x\in\Xcal_p\) とする。各ノードの局所入力は入力構成写像で構成される：

\[ \iota_{p,v}:\left(\prod_{u\in\Pa_p(v)}\Scal_{\tau_p(u)}\right)\times \Xcal_p \to \Zcal_{\tau_p(v)}. \]

ここで \(\Zcal_j\) は操作型 \(M_j\) の局所入力空間、\(\Scal_j\) はその操作後の潜在状態空間である。

プロトコル実行過程

各操作型の真の写像

各モジュール型 \(M_j\) に未知の真の写像 \(F_j^\dagger:\Zcal_j\to\Scal_j\) が対応する。同じモジュール型が異なるプロトコルに現れる場合、そのモジュールは同じ操作的意味論、すなわち同じ \(F_j^\dagger\) に従うと仮定する。

外部入力と再帰実行

設計変数 \(x\) が与えられると、各ノードに外部入力 \(u_{p,v}(x)\in\Ucal_{\tau_p(v)}\) が対応する（外部入力を持たなければ一点集合）。DAG の位相順序に従い真の出力は

\[ z_v^\dagger(x) = \iota_{p,v}\!\left((s_u^\dagger(x))_{u\in\Pa_p(v)},\, u_{p,v}(x)\right),\qquad s_v^\dagger(x) = F_{\tau_p(v)}^\dagger\!\left(z_v^\dagger(x)\right). \]
測定器と観測可能ノード

測定器として登録された操作型の添字集合を \(\Qcal\subseteq\{1,\ldots,m\}\) とする。観測可能ノード集合は

\[ \Obs(p)=\{v\in V_p:\tau_p(v)\in\Qcal\}. \]

任意の中間ノード出力が直接観測できるとは仮定しない。観測できるのはプロトコル中に明示的に含まれる測定器モジュールの出力のみである。

観測空間と観測データ

観測空間を \(\Ycal_p^{\mathrm{obs}}=\prod_{q\in\Obs(p)}\Scal_{\tau_p(q)}\)、真の観測出力を \(\Eval^\dagger(p;x)=\left(s_q^\dagger(x)\right)_{q\in\Obs(p)}\) とする。観測データは

\[ \Dcal_n=\{(p_i,x_i,y_i)\}_{i=1}^n,\qquad y_i = \Eval^\dagger(p_i;x_i)+\xi_i, \]

\(\xi_i\) は観測空間 \(\Ycal_{p_i}^{\mathrm{obs}}\) 上の観測ノイズ。

主設定：terminal-only observation

本研究の主設定では各プロトコルで \(\Obs(p)=\{q_p^{\mathrm{term}}\}\)、観測データは終端測定器ノードの出力のみを含む。途中段階で観測を追加する場合も、中間状態そのものではなく、その位置に置いた測定器モジュールの出力を観測するものとして表す。この点で OFML は、中間ノード出力を直接利用する function network 型ベイズ最適化[bofn]や、ノード単位の部分評価を前提とする pKGFN[pKGFN]とは異なる。

Source-to-target protocol transfer

source と target

初期データ \(\Dcal_n\) は主に source \(\Pcal_{\mathrm{src}}\) から得られる。転移・予測・追加実験の対象を target \(\Pcal_{\mathrm{tar}}\subseteq\Pcal\) と書く（一つに限らない — 観測に応じて次に試すプロトコルが変わりうるため）。典型的な zero-shot transfer では \(\Pcal_{\mathrm{tar}}\cap\Pcal_{\mathrm{src}}=\emptyset\)。

target 関数と転移目的

各 \(p\in\Pcal_{\mathrm{tar}}\) の真の target 関数を \(f_p^\dagger(x)=\Eval^\dagger(p;x)\) とする。目的は source の観測と（必要に応じて）少数の target 観測から \(\{f_p^\dagger:p\in\Pcal_{\mathrm{tar}}\}\) を予測すること。データは \(\Dcal_n=\Dcal_{\mathrm{src}}\cup \Dcal_{\mathrm{tar},n}\) と分け、zero-shot では \(\Dcal_{\mathrm{tar},n}=\emptyset\)、few-shot / active correction では逐次増える。

Closed-loop target correction

候補実験と観測更新

追加実験が許されるとき、候補集合 \(\Ccal_n\subseteq\{(p,x):p\in\Pcal_{\mathrm{tar}}\}\) から \(c_n=(p_n,x_n)\) を選び、観測 \(y_n=\Eval^\dagger(p_n;x_n)+\xi_n\) を得て \(\Dcal_{n+1}=\Dcal_n\cup\{(p_n,x_n,y_n)\}\) と更新する。本論文の実験では主張を明確にするため、各エピソードで単一の target \(\Pcal_{\mathrm{tar}}=\{p_\star\}\) を固定して評価する。

学習目的

第一の目的：target 上の予測誤差

任意の予測器 \(\widehat f_n\) に対し、target 上の予測誤差を

\[ \mathcal E_{\mathrm{tar}}(\widehat f_n) = \left( \frac{1}{|\Pcal_{\mathrm{tar}}|}\sum_{p\in\Pcal_{\mathrm{tar}}} \E_{x\sim\nu_p}\!\left[\,\norm{\widehat f_n(p,x)-f_p^\dagger(x)}^2\right] \right)^{1/2} \]

と定義する（\(\nu_p\) は target \(p\) 上の評価分布）。\(\Pcal_{\mathrm{tar}}=\{p_\star\}\) なら単一 target 上の予測誤差に帰着する。

第二の目的：予算内で誤差を効率よく下げる

限られた追加実験予算のもとで \(\mathcal E_{\mathrm{tar}}(\widehat f_n)\) を効率よく下げること。これが 能動学習の評価軸となる。

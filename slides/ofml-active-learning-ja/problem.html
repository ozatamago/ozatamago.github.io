Problem setting

終端観測付きプロトコル構造化代理モデリング問題

OFML が解く対象を定義する。プロトコル構造・真の潜在実行・観測モデル・OFML 仮説クラス・source→target 学習・逐次行動とコスト指標目的を要点ごとに箱で示す（地の文は補足のみ）。合成の重み共有アーキテクチャの詳細は Object-Flow Machine Learning、終端関数事後の一致性は Theoretical statement and proofs で扱う。記号は実験のコスト指標評価および理論と整合させる。

3.1 プロトコル語彙と型付き実行グラフ

操作語彙とプロトコル集合

共通する操作語彙を \(\Mcal=\{M_1,\ldots,M_m\}\) とする。各 \(M_j\) は分注・混合・加熱・培養・反応・読み出し準備など、プロトコル間で再利用される操作型である。プロトコル集合を \(\Pcal\)、観測済みの source を \(\Pcal_{\mathrm{src}}\subseteq\Pcal\) とする（transfer 先の有限 target 集合は 3.4 で定義）。

型付き有向非巡回グラフ（DAG）と局所入力構成写像

各プロトコル \(p\in\Pcal\) は型付き DAG \(p=(V_p,E_p,\tau_p,\iota_p)\)。\(V_p\) はノード集合、\(E_p\) は依存関係、\(\tau_p:V_p\to\{1,\ldots,m\}\) は各ノードに操作型を割り当てる。ノード \(v\) の親集合を \(\Pa_p(v)\)、外部設計変数を \(x\in\Xcal_p\) とする。操作型 \(M_j\) の局所入力空間を \(\Zcal_j\)、操作後の潜在状態空間を \(\Scal_j\) とし、各ノードの局所入力を組み立てる入力構成写像を一度だけ次で定める：

\[ \iota_{p,v}:\left(\prod_{u\in\Pa_p(v)}\Scal_{\tau_p(u)}\right)\times \Xcal_p \to \Zcal_{\tau_p(v)}. \]
真の潜在実行（プロトコル依存の真写像）

各操作型 \(M_j\) の真の写像を、当該プロトコルに依存して \(F_{j,p}^\dagger:\Zcal_j\to\Scal_j\) と書く。DAG の位相順序に従い、設計変数 \(x\) から真の状態は次で定まる：

\[ z_v^\dagger(x) = \iota_{p,v}\!\left((s_u^\dagger(x))_{u\in\Pa_p(v)},\, x\right),\qquad s_v^\dagger(x) = F_{\tau_p(v),\,p}^\dagger\!\left(z_v^\dagger(x)\right). \]

※ 真写像をプロトコル依存 \(F_{j,p}^\dagger\) としたのは意図的である。OFML のモデルは同一操作型に単一モジュール \(g_{\theta_j}\) を共有する（3.3）が、真実がその共有と一致するかは転移が成立するための条件であって前提ではない（3.4 の C2、仮定 8）。意味論シフト s2 はこの条件を意図的に破る失敗モードである。

3.2 観測モデル

観測器語彙と観測可能ノード

測定器を操作語彙とは別の観測器語彙 \(\Qcal=\{Q_1,\ldots,Q_r\}\) として分離する。各観測器 \(Q_q\) は、それが置かれたノードの潜在状態を読み出す真の読み出し写像 \(h_q^\dagger:\Scal_{j}\to\Ycal_q\) を持つ。プロトコル \(p\) の観測可能ノード集合を \(\Obs(p)\subseteq V_p\)（観測器が置かれたノード）とする。任意の中間ノード出力が直接観測できるとは仮定せず、観測できるのは明示的に置かれた観測器の出力のみである。

※ 観測器を操作語彙から分けることで、将来の複数観測器・観測器ごとのコスト・中間測定・非破壊/破壊測定の拡張が自然になる。本論文の主設定では \(\Qcal=\{q_{\mathrm{term}}\}\)、\(\Ycal_{q_{\mathrm{term}}}=\R\)（単一の終端スカラー観測器）。

観測空間と雑音観測

観測空間を \(\Ycal_p^{\mathrm{obs}}=\prod_{q\in\Obs(p)}\Ycal_q\)、真の観測出力を \(\Eval^\dagger(p;x)=\left(h_q^\dagger(s_{v(q)}^\dagger(x))\right)_{q\in\Obs(p)}\) とする（\(v(q)\) は観測器 \(q\) が置かれたノード）。観測データは

\[ y_i = \Eval^\dagger(p_i;x_i)+\xi_i,\qquad \E[\xi_i\mid \cdot]=0, \]

\(\xi_i\) は観測空間 \(\Ycal_{p_i}^{\mathrm{obs}}\) 上の条件付き不偏・有限分散雑音（理論の 仮定 4）。

主設定：terminal-only observation

本研究の主設定では各プロトコルで \(\Obs(p)=\{q_p^{\mathrm{term}}\}\)、観測データは終端測定器ノードの出力（単一スカラー \(y\)）のみを含む。学習中に途中段階を観測する場合も、中間状態そのものではなく、その位置に置いた観測器モジュールの出力として表す。この点で OFML は、中間ノード出力を直接利用する function network 型ベイズ最適化[bofn]や、ノード単位の部分評価を前提とする pKGFN[pKGFN]とは異なる。

3.3 OFML 仮説クラス

共有操作モジュールと合成予測器

各操作型 \(M_j\) に学習される共有モジュール \(g_{\theta_j}:\Zcal_j\to\Scal_j\) を割り当てる。観測器 \(Q_q\) の読み出しヘッド \(h_q\) も学習対象とし、全学習成分（操作モジュール＋観測器ヘッド）を \(\theta\in\Theta\) と束ねる（主設定の終端読み出しは固定のスカラー測定、1-C の中間観測器ヘッドは学習可能）。真の実行と同じ位相順序で合成し、予測器を

\[ \Eval(p;\theta;x)=\bigl(h_q(s_{v(q)}^{\theta}(x))\bigr)_{q\in\Obs(p)},\quad s_v^{\theta}(x)=g_{\tau_p(v)}\!\bigl(\iota_{p,v}((s_u^{\theta}(x))_{u\in\Pa_p(v)},x);\theta\bigr) \]

と定める。同一操作型に単一パラメータ \(\theta_j\) を共有することが OFML の帰納バイアスであり、これが順序・出現回数（count）を無視する非構造ベースライン（Shared-Ens／Independent-Ens）との違いを与える。合成アーキテクチャの詳細は Object-Flow Machine Learning。

パラメータ不確実性と終端関数への pushforward

モジュールパラメータの不確実性分布を \(\Pi_n^\theta\) とする。これは exact posterior・SMC・deep ensemble を同一視するものではなく、いずれも共通して提供する「終端関数サンプル」のインターフェースを定義するためのものである。各 target \(p\) について pushforward \(T_p(\theta)(x)=\Eval(p;\theta;x)\) により終端関数の分布

\[ \Pi_n^{G}=(T_p)_{\#}\,\Pi_n^\theta,\qquad \widehat f_{n,p}(x)=\mu_{n,p}(x)=\E_{\theta\sim\Pi_n^\theta}\!\left[\Eval(p;\theta;x)\right] \]

を得る。予測はこの事後平均 \(\mu_{n,p}\)、能動学習の獲得はこの分布の分散に基づく。\(\Pi_n^{G}\) は理論が一致性を論じる \(C(\Xcal)\) 値事後 \(\Law(G\mid\Hcal_n)\)（補題 5）に対応する。

3.4 Source-to-target 学習

source / target データの分離

データを source と target に分ける：

\[ \Dcal_n=\Dcal_{\mathrm{src}}\cup\Dcal_{\mathrm{tar},n},\qquad \Dcal_{\mathrm{tar},0}=\emptyset. \]

\(\Dcal_{\mathrm{src}}\) は target のコスト計上より前に固定された source 観測、\(\Dcal_{\mathrm{tar},n}\) は逐次追加される target 観測である。\(n=0\) は source 事前学習後・target 観測なし（zero-shot 転移）、\(n\ge1\) は target 適応に対応する。source 取得費は全モデル共通の sunk cost として扱い、コスト軸には含めない。

有限 target 集合と転移目的

転移・評価の対象を有限 target プロトコル集合 \(\Pcal_\star=\{p_1,\ldots,p_K\}\subseteq\Pcal\) とする（典型的に \(\Pcal_\star\cap\Pcal_{\mathrm{src}}=\emptyset\)）。各 \(p\in\Pcal_\star\) の真の target 関数は \(f_p^\dagger(x)=\Eval^\dagger(p;x)\)。目的は source と（必要に応じて）少数の target 観測から \(\{f_p^\dagger:p\in\Pcal_\star\}\) を予測することである。

※ 単一 target は \(K=1\) の特殊例。本論文の Figure 2 は \(\Pcal_\star=\{\mathrm{ABO},\mathrm{BAO}\}\)（あるいは \(\{\mathrm{ABCMO},\mathrm{CBAMO}\}\)）の 2 target を同時評価する。2 target を用いることで、順序を無視する Shared-Ens の記述子エイリアシング床 \(F_{\mathrm{alias}}\)（コスト指標）が定義できる。

転移条件 C2（意味論の共有）

OFML のモデルは常に \(g_{\theta_j}\) を共有するが、転移が成立するのは真実がその共有と一致するときである。共有意味論条件を

\[ F_{j,p}^\dagger=F_j^\dagger\quad\text{(relevant }p\text{ について)} \]

と条件 C2 で表す。これが破れると（意味論シフト s2）zero-shot 転移は失敗しうる。転移可否の完全な診断 C1–C5 は 仮定 8、線形の rank 条件は 命題 4 を参照。

3.5 逐次行動とコスト指標目的

履歴・行動・方策

観測履歴を \(\Hcal_n=\sigma(\Dcal_{\mathrm{src}},\Dcal_{\mathrm{tar},n})\)、行動を \(a_n=(p_n,x_n,q_n)\)（対象プロトコル・設計点・観測器）とし、次の行動は履歴に基づいて選ばれる：\(a_{n+1}=\pi_n(\Hcal_n)\)。候補集合は

\[ \Acal_n\subseteq\bigsqcup_{p\in\Pcal_\star}\{p\}\times\Xcal_p \]

（実験では有限 pool、理論では compact 連続領域）。本論文の主設定では \(p_n\in\Pcal_\star\)、\(q_n=q_{\mathrm{term}}\)、単位コスト \(c(a_n)=1\)。これは理論の \(\Pi_n=\Law(G\mid\Hcal_n)\)（定理 3）と接続する。

累積 target コストと予算

累積 target 取得コストを

\[ B_n=\sum_{i=1}^{n}c(a_i),\qquad \text{単位コストのとき } B_n=n,\quad B_0=0. \]

\(B_0=0\) は source 事前学習費を含まない条件付きの target コストである（source は全モデル共通の sunk cost）。

コスト指標の学習目的（統一）

プールドスケール \(s_\star=\sqrt{\sum_{p\in\Pcal_\star} w_p\,\mean((f_p^\dagger-\bar f)^2)}\)（等重み \(w_p=1/K\)）で正規化した、累積コスト \(b\) におけるマクロ target 誤差を

\[ \varepsilon_m(b)=\left(\sum_{p\in\Pcal_\star} w_p\;\E_{x\sim\nu_p}\!\left[\tfrac{\norm{\mu_{n,p}(x)-f_p^\dagger(x)}^2}{s_\star^2}\right]\right)^{1/2},\qquad \bar J(B)=\frac1B\int_0^B\varepsilon_m(b)\,db \]

と定義する（\(\nu_p\)＝target \(p\) 上の評価分布、実験では有限 test set で近似、\(\bar J\) は台形則）。OFML の目的は予算 \([0,B]\) 全体で \(\varepsilon_m(b)\) を小さくすることに統一される：\(\varepsilon_m(0)\) は順方向転移（\(b=0\)）、\(\varepsilon_m(b),\,b>0\) はtarget 適応を測り、\(\bar J(B)\) と端点 \(\varepsilon_m(0)/\varepsilon_m(B)\) を併記する。
